|------|------|----------|----------|----------|

## 切换方案

所有方案都使用相同的配置文件，切换非常简单：

### 从Server酱切换到企业微信

```ini
[Notification]
enabled = true
# push_type = serverchan  # 注释掉
push_type = qywechat      # 改为企业微信
# serverchan_key = SCT...  # 保留旧配置
qywechat_webhook = https://...  # 添加新配置
```

### 从任意方案切换到公众号

```ini
[Notification]
enabled = true
push_type = wechat_official  # 改为公众号
wechat_appid = wx...
wechat_secret = ...
wechat_template_id = ...
wechat_openids = ...
```

## 常见问题

### Q: 可以同时使用多个方案吗？

可以在代码中实现，但配置文件目前只支持一种。

建议：选择一个主要方案，其他作为备份。

### Q: 哪个方案最稳定？

稳定性排序：
1. 微信公众号（官方）
2. 企业微信（官方）
3. Server酱（第三方）
4. PushPlus（第三方）

### Q: 如何选择？

简单决策树：

```
是企业用户? 
  是 → 微信公众号
  否 → 
    需要多人接收?
      是 → 企业微信
      否 → Server酱
```

### Q: 推送失败怎么办？

1. 查看日志：`logs/notification.log`
2. 运行测试：`python test_push.py`
3. 检查配置是否正确
4. 尝试切换到其他方案
